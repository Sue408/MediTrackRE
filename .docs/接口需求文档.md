# Meditrack 后端接口需求文档

## 概述

本文档定义了Meditrack药品追踪管理系统所需的后端API接口需求。系统采用现代化的卡片式布局设计，遵循Auth.vue的视觉风格。

---

## 一、用户相关接口

### 1.1 获取当前用户信息

**接口路径**: `GET /api/user/info`

**请求头**:
```
Authorization: Bearer <access_token>
```

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "用户ID",
    "nickname": "用户昵称",
    "email": "user@example.com",
    "avatar": "头像URL",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

### 1.2 更新用户信息

**接口路径**: `PUT /api/user/info`

**请求头**:
```
Authorization: Bearer <access_token>
```

**请求体**:
```json
{
  "nickname": "新昵称",
  "avatar": "头像URL（可选）"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

### 1.3 修改密码

**接口路径**: `PUT /api/user/password`

**请求头**:
```
Authorization: Bearer <access_token>
```

**请求体**:
```json
{
  "old_password": "旧密码",
  "new_password": "新密码"
}
```

## 二、药品管理接口

### 2.1 获取药品列表

**接口路径**: `GET /api/medicines`

**请求头**:
```
Authorization: Bearer <access_token>
```

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认10 |
| keyword | string | 否 | 搜索关键词（药品名称） |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "药品ID",
        "name": "药品名称",
        "specification": "规格",
        "manufacturer": "生产厂家",
        "unit": "单位",
        "category": "药品分类",
        "image_url": "药品图片URL",
        "description": "说明",
        "created_at": "2024-01-01T00:00:00Z"
      }
    ],
    "total": 100,
    "page": 1,
    "page_size": 10
  }
}
```

### 2.2 获取药品详情

**接口路径**: `GET /api/medicines/:id`

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "药品ID",
    "name": "药品名称",
    "specification": "规格",
    "manufacturer": "生产厂家",

    "description": "说明",
    "usage": "用法用量",

    "adverse_reactions": "不良反应",
    "storage_conditions": "储存条件",

  }
}
```

### 2.3 添加药品

**接口路径**: `POST /api/medicines`

**请求体**:
```json
{
  "name": "药品名称",
  "specification": "规格",
  "manufacturer": "生产厂家",
  "unit": "单位",
  "category": "药品分类",
```

**接口路径**: `PUT /api/medicines/:id`

### 2.5 删除药品

**接口路径**: `DELETE /api/medicines/:id`

## 三、计划管理接口

### 3.1 获取用药计划列表

**接口路径**: `GET /api/plans`

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | string | 否 | 日期（YYYY-MM-DD），默认今天 |
| status | string | 否 | 状态：pending/completed/all |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "计划ID",
        "medicine_id": "药品ID",
        "medicine_name": "药品名称",
        "medicine_image": "药品图片",
        "dosage": "剂量",
        "unit": "单位",
        "scheduled_time": "计划时间",
        "status": "pending",
        "actual_time": null,
        "notes": "备注",
        "created_at": "2024-01-01T00:00:00Z"
      }
    ],
    "date": "2024-01-01"
  }
}
```

### 3.2 创建用药计划

**接口路径**: `POST /api/plans`

**请求体**:
```json
{
  "medicine_id": "药品ID",
  "dosage": "剂量",
  "scheduled_time": "计划时间（HH:mm）",
  "repeat_type": "重复类型：once/daily/weekly/monthly",
  "repeat_days": "重复日期（weekly: 1-7, monthly: 1-31）",
  "notes": "备注（可选）",
  "start_date": "开始日期",
  "end_date": "结束日期（可选）"
}
```

### 3.3 更新用药计划

**接口路径**: `PUT /api/plans/:id`

### 3.4 删除用药计划

**接口路径**: `DELETE /api/plans/:id`

### 3.5 记录用药

**接口路径**: `POST /api/plans/:id/take`

**请求体**:
```json
{
  "actual_time": "实际服用时间",
  "status": "completed",
  "notes": "备注（可选）"
}
```

### 3.6 获取用药统计

**接口路径**: `GET /api/plans/statistics`

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| start_date | string | 是 | 开始日期 |
| end_date | string | 是 | 结束日期 |

**响应示例**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total_plans": 100,
    "completed": 85,
    "missed": 10,
    "skipped": 5,
    "compliance_rate": 85.0,
    "daily_stats": [
      {
        "date": "2024-01-01",
        "total": 10,
        "completed": 8,
        "missed": 2
      }
    ]
  }
}
```

## 五、通知接口

### 5.1 获取通知列表

**接口路径**: `GET /api/notifications`

**查询参数**:
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 否 | 类型：reminder/system/all |
| is_read | bool | 否 | 已读/未读 |
| page | int | 否 | 页码 |
| page_size | int | 否 | 每页数量 |

### 5.2 标记通知为已读

**接口路径**: `PUT /api/notifications/:id/read`

### 5.3 标记所有通知为已读

**接口路径**: `PUT /api/notifications/read-all`

### 5.4 删除通知

**接口路径**: `DELETE /api/notifications/:id`

## 六、通用响应格式

### 成功响应
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

### 错误响应
```json
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

### 常见错误码
| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权（token无效或过期） |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 七、接口设计原则

1. **RESTful风格**: 使用标准HTTP方法（GET/POST/PUT/DELETE）
2. **认证方式**: JWT Token（Bearer Token）
3. **分页**: 采用limit/offset或page/page_size方式
4. **时间格式**: ISO 8601格式（YYYY-MM-DDTHH:mm:ssZ）
5. **图片URL**: 使用完整的绝对路径
6. **卡片式数据**: 列表接口返回的数据应便于前端展示为卡片组件
