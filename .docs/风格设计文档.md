# 应用风格约束文档

## 核心配色方案

### 主色调
- **品牌色**：`#7494ec` (柔和蓝紫色)
- **辅助背景**：线性渐变 `linear-gradient(90deg, #e2e2e2, #c9d6ff)`
- **纯净背景**：`#fff` (白色)

### 文字颜色
- **主标题/正文**：`#333` (深灰)
- **高亮/按钮文字**：`#fff` (白色)

### 按钮状态
- **默认**：透明背景 + 白色边框 (`border: 2px solid #fff`)
- **悬停**：`background-color: rgba(255, 255, 255, 0.2)` (半透明白色)
- **无阴影原则**：按钮不添加阴影，保持扁平化

## 布局与容器

### 容器规范
- **大容器**：圆角 `30px`，阴影 `box-shadow: 0 0 30px rgba(0, 0, 0, 0.2)`
- **内边距**：默认 `40px` (响应式可缩减至 `20px`)
- **居中方式**：`flex` 居中 (`align-items: center; justify-content: center`)

### 表单区域
- **最小高度**：`100vh` 确保全屏覆盖
- **内容区域**：`width: 750px; height: 450px` (桌面端基准)
- **溢出处理**：`overflow: hidden` 保持视觉整洁

## 动画与过渡

### 时间曲线
- **平滑切换**：`ease-in-out`
- **主动画时长**：`1.8s` (用于背景变换)
- **次要动画**：`0.6s` (用于面板移动)

### 延迟策略
- **交互动效**：采用 `transition-delay` 创造层次感
- **左面板**：延迟 `1.2s`
- **右面板**：延迟 `0.6s`
- **状态切换时**：延迟顺序互换，创造"接力"效果

## 响应式设计

### 断点规范
- **移动优先**：`max-width: 650px`
- **小屏优化**：`max-width: 400px`

### 移动端适配策略
- **垂直分层**：表单占 `70%` 高度，切换面板占 `30%`
- **背景变形**：从横向移动改为纵向移动 (`top` 属性控制)
- **圆角调整**：使用 `vw` 单位 (`border-radius: 20vw`)
- **位置重置**：`left: 0; right: 0` 简化定位

### 内容适配
- **字体缩放**：标题在超小屏时缩减至 `30px`
- **内边距**：小屏时从 `40px` 缩减至 `20px`

## 交互细节

### 视觉层次
- **Z-index 管理**：
  - 背景层：`z-index: 2`
  - 内容层：自然堆叠
- **指针事件**：切换面板背景 `pointer-events: none`，按钮单独启用

### 状态反馈
- **错误提示**：通过按钮状态传递 (`button-error` 属性)
- **加载状态**：通过 `loading` 属性控制
- **错误清除**：自动延迟清除 (1秒后)

## 设计哲学总结

1. **柔和渐进**：像剥开橙子一样，一层层展开界面
2. **呼吸感空间**：足够的留白和圆角，避免尖锐
3. **色彩克制**：主色+白+灰，保持专注
4. **有节奏的动效**：像果汁流动一样自然顺畅
5. **触碰友好**：移动端优先，按钮大小适中
6. **错误温柔提示**：不惊吓用户，安静地反馈

---

**适用范围**：所有认证页面、模态框、侧边栏切换等需要平滑状态转换的组件
**设计系统命名**：建议命名为 "Flow UI System" (流动式界面系统)
