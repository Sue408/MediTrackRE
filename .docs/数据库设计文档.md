# 后端数据库设计文档

## 1 数据表设计

### 1.1 用户表（users）
- id <Integer>: 主键
- nickname <String>: 昵称，非空
- email <String>: 邮箱，唯一，非空
- password_hash <String>: 密码哈希
- avatar <Text>: 头像 (base64编码 or URL)
- is_active <Boolean>: 用户激活状态
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间
- updated_at <DateTime>: 更新时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.2 健康档案表 (profiles)
- id <Integer>: 主键
- user_id <Integer>: 外键，关联用户主键，唯一，非空
- gender <String>: 性别，枚举 (男/女/unknown)，默认值为'unknown'
- height <Decimal>: 身高 (单位:cm)
- weight <Decimal>: 体重 (单位:kg)
- blood_type <String>: 血型 (A/B/AB/O/unknown)，默认值为'unknown'
- allergies <JSON>: 过敏史，每项包含'name'、'reaction'字段，默认值采用`default=dict`生成空字典
- diseases <JSON>: 疾病史，每项包含'name'、'diagnosed_at'、'status'字段，默认值采用`default=dict`生成空字典
- special_status <String|None>: 女性用户特殊状态，枚举 ('备孕'/'怀孕'/'哺乳'/none)
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间
- updated_at <DateTime>: 更新时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.3 药品信息表 (drugs)
- id <Integer>: 主键
- user_id <Integer>: 外键，关联用户主键，非空
- name <String>: 药品名称，非空
- image <Text>: 药品图片 (Base64编码 or URL)
- usage_method <String>: 用法，非空，枚举 (口服/注射/外用/...)
- dosage <String>: 用量，非空 (如'2粒')
- remaining <String>: 余量，可空 (如'24粒')
- source <String>: 来源，非空，枚举 ('医疗数据库'/'手动输入')
- external_id <Integer>，外部数据库ID，可空
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间
- updated_at <DateTime>: 更新时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.4 用户计划表 (plans)
- id <Integer>: 主键
- user_id <Integer>: 外键，关联用户主键，非空
- name <String>: 计划名称，非空
- plan_type <String>: 计划类型，非空，枚举 (长期/周期)
- start_date <DateTime>: 开始时间，非空
- end_date <DateTime>: 结束时间，长期类型此项为空
- frequency_type <String>: 用药频率类型，非空，枚举 (daily/alternate/weekly/monthly)
- frequency_detail <JSON>: 用药频率详情，daily频率类型此项为空，(如'间隔天数/星期几/每月几号')
- time_points <JSON>: 每日用药时间点数组 (如`['8:00'、'20:00']`)
- sms_enabled <Boolean>: 是否启用短信提醒服务
- status <String>: 计划状态，非空，枚举 (active/paused/completed)
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间
- updated_at <DateTime>: 更新时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.5 历史用药记录表 (tracks)
- id <Integer>: 主键
- user_id <Integer>: 外键，关联用户主键，非空
- plan_id <Integer>: 外键，关联计划主键，非空
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.6 用户关系映射表 (relations)
- id <Integer>: 主键
- user_id <Integer>: 外键，非空，关联发起方
- target_user_id <Integer>: 外键，非空，关联目标方
- relation_type <String>: 目标方相对于发起方的关系类型，非空，枚举 ('患者'/'监护人')
- status <String>: 关系状态，非空，枚举 (pending/accepted/rejected)
- remark <String>: 备注名称，非空，默认为目标方的昵称
- created_at <DateTime>: 创建时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间
- updated_at <DateTime>: 更新时间，默认采用`default=datetime.now(timezone.utc)`获取UTC时间

### 1.7 计划药品中间表 (plan_drugs)
- id <Integer>: 主键
- plan_id <Integer>: 外键，关联计划主键，非空
- drug_id <Integer>: 外键，关联计划主键，非空
- dosage <String>: 计划内的药品用量，非空，可与药品的默认用量不同，默认采用药品默认用量
